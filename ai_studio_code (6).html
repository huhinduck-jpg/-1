마크다운 기반 슬라이드 제작

Pandoc, Reveal.js, Marp 활용 워크플로우 교육

교육 목표: 제작 시간을 단축하고 일관성 있는 발표 자료 생성

교육 대상: Markdown을 사용하는 연구원, 개발자, 실무자

본 교육은 마크다운(Markdown)이라는 단순한 텍스트 문법을 활용하여 전문가 수준의 발표 자료를 신속하게 제작하는 방법을 다룹니다. 특히 Pandoc과 Reveal.js, Marp와 같은 강력한 도구들을 연동하여 작업 효율을 극대화하는 실질적인 방법을 학습하는 데 중점을 둡니다.

Markdown 슬라이드 도입 배경

기존 PPT 도구의 한계점과 효율적 대안

기존 도구의 한계

  • 레이아웃 조정에 많은 시간 소요
  • 코드 포함 시 글꼴/레이아웃 문제
  • 콘텐츠보다 디자인에 집중하게 됨

마크다운의 장점

  • 콘텐츠 자체에 집중하여 생산성 향상
  • 선명한 코드 및 하이라이팅 지원
  • 텍스트 기반으로 버전 관리에 용이
파워포인트나 구글 슬라이드 같은 전통적인 도구는 시각적 완성도는 높지만, 특히 코드가 포함된 자료를 만들 때 레이아웃과 글꼴 문제로 시간이 많이 소요됩니다. 마크다운은 이러한 문제를 해결하며, 콘텐츠 중심의 빠른 작성 환경을 제공하여 슬라이드 제작 과정을 혁신할 수 있습니다.

주요 Markdown 슬라이드 도구

핵심 도구 Marp, Reveal.js, Pandoc의 역할

Marp

VS Code 확장으로 실시간 미리보기를 제공하는 가장 간편한 도구

Reveal.js

HTML 기반의 강력한 프레젠테이션 프레임워크. 다양한 효과 지원

Pandoc

마크다운을 PPTX, HTML 등 수많은 형식으로 변환하는 만능 변환기

Marp는 단순하고 빠른 슬라이드 작성이 목표일 때 유용하며, 특히 VS Code 확장으로 실시간 미리보기를 지원합니다. 반면, Pandoc은 마크다운을 PPTX나 Beamer 같은 다양한 포맷으로 변환하는 데 사용되는 범용 변환 엔진이며, Reveal.js는 HTML 슬라이드에 고급 애니메이션과 코드 강조 기능을 제공하는 프레임워크입니다.

도구별 주요 특징 및 비교

Marp와 Pandoc+Reveal.js의 장단점 요약

도구/결합 주요 포맷 레이아웃 유연성 코드 하이라이팅 비고
Marp PDF, PPTX, HTML 낮음 (제한적) 우수 (기본 지원) 빠르고 간편한 작성
Pandoc + PPTX DOCX, PPTX 중간 (참조 문서 사용) 지원 (내부 엔진) 기존 워드 프로세서와 호환
Pandoc + Reveal.js HTML 슬라이드 높음 (CSS/JS) 최상 (Highlight.js) 다양한 웹 기반 효과
Marp는 배우기 쉽고 빠르지만, 복잡한 레이아웃이 필요할 때 파워포인트로 돌아가고 싶게 만들 수 있습니다. 반면, Pandoc은 다양한 확장 문법과 포맷 변환 옵션을 제공하여 학술 자료 제작에 적합합니다. 특히 Reveal.js와 결합하면 코드 하이라이팅이나 단계별 표시 같은 고급 기능을 활용할 수 있습니다.

Pandoc 슬라이드 변환 흐름

마크다운 파일을 발표 자료로 만드는 기본 단계

Markdown 파일
(input.md)
➔
Pandoc 실행
($ pandoc -t pptx ...)
➔
추상 구문 트리 (AST)
(내부 표현)
➔
/ 출력
(output.pptx / .html)

하나의 소스로 다양한 결과물을 만들어내는 것이 핵심입니다.

Pandoc의 작동 원리는 간단합니다. 입력 포맷(마크다운)을 내부의 추상 구문 트리(AST)로 파싱한 다음, 이를 원하는 출력 포맷(PPTX, Reveal.js 등)으로 변환하는 것입니다. 슬라이드를 만들 때는 반드시 출력 포맷을 지정해야 하며, --standalone 옵션을 잊지 않아야 합니다.

슬라이드 구조 및 분할

마크다운 구분자를 이용한 페이지 나누기

Reveal.js는 수평/수직 슬라이드를 모두 지원합니다.

아래 화살표를 눌러 수직 슬라이드를 확인해보세요.

마크다운에서 슬라이드를 나누는 가장 간단한 방법은 수평 분할자(`---`)를 사용하는 것입니다. Reveal.js를 사용하면, 수평(`---`)과 수직(`--`) 구분자를 활용하여 2차원적인 슬라이드 흐름을 구성할 수 있습니다. Pandoc은 기본적으로 제목 레벨을 분석하여 슬라이드를 분할합니다.

수평 분할 (`---`)

가장 일반적인 슬라이드 분할 방식으로, 다음 섹션으로 이동합니다. (좌-우 이동)


# 첫 번째 슬라이드
내용...
---
# 두 번째 슬라이드
내용...

수직 분할 (`--` 또는 `----`)

동일한 주제 내에서 하위 슬라이드를 생성합니다. (상-하 이동)


# 메인 주제
---
## 하위 주제 1
--
## 하위 주제 2

PPTX 출력을 위한 Pandoc의 가장 강력한 기능 중 하나는 `--reference-doc`입니다. 이를 통해 회사의 표준 템플릿 레이아웃과 디자인을 새로운 마크다운 기반 슬라이드에 쉽게 적용할 수 있습니다. 참조 문서에 정의된 레이아웃 이름은 Pandoc이 슬라이드 콘텐츠에 따라 레이아웃을 선택하는 기준이 됩니다.

코드 하이라이팅 적용 (Live Demo)

Reveal.js와 Pandoc을 이용한 깔끔한 코드 제시

data-line-numbers="3, 8-10" 속성을 사용하여 특정 줄을 강조할 수 있습니다.


def calculate_fib(n):
    a, b = 0, 1
    for _ in range(n):  # 이 줄은 3번 줄입니다.
        yield a
        a, b = b, a + b
    # ... 중간 생략 ...
    # 8-10줄을 강조하려면
    if n > 100:
        print("Done")
    return
기술 발표에서 코드 표시는 매우 중요합니다. Reveal.js는 `highlight.js` 기반으로 강력한 구문 강조 기능을 제공하며, 펜스드 코드 블록 내의 속성(예: `data-line-numbers`)을 통해 특정 줄을 강조하거나 줄 번호를 표시할 수 있습니다. Pandoc은 `-t revealjs`로 변환할 때 이러한 속성을 HTML에 반영합니다.

단계별 표시 (Fragments) - Live Demo

콘텐츠를 순차적으로 보여주는 애니메이션 기법

  • 첫 번째 요소는 커지면서 나타납니다. (grow)
  • 두 번째 요소는 파란색으로 강조됩니다. (highlight-blue)
  • 세 번째는 나타났다가 사라집니다. (fade-in-then-out)
  • 마지막은 아래에서 위로 나타납니다. (fade-up)
프래그먼트는 발표 중 청중의 집중을 유도하고 내용을 단계적으로 설명하는 데 필수적입니다. Reveal.js에서 이 기능은 `fragment` 클래스를 사용하여 구현됩니다. Pandoc으로 변환된 HTML 슬라이드에서 `` 태그나 글머리 기호에 이 클래스를 적용할 수 있습니다.

코드 블록 단계별 강조 (Live Demo)

| 구분자를 활용한 코드 프래그먼트

다음 버튼을 눌러보세요. 단계별로 다른 코드 라인이 강조됩니다.


# 1단계: 3~5줄 강조
def setup():
    initialize_config()
    load_plugins()

# 2단계: 8~10줄 강조
def run_analysis():
    data = fetch_data()
    process_data(data)
    save_results(data)

# 3단계: 13~15줄 강조
if __name__ == "__main__":
    setup()
    run_analysis()
기술 발표 시 복잡한 코드를 설명할 때, 코드 단계별 강조 기능은 매우 유용합니다. `data-line-numbers` 속성에 줄 번호를 지정하되, 단계별 구분을 위해 파이프(`|`)를 사용합니다. 이 기능은 Reveal.js의 `highlight` 플러그인에 의해 제공되며, Pandoc 변환 시 속성이 올바르게 적용되어야 합니다.

이 워크플로우는 마크다운 작성부터 웹 기반 발표 자료 배포까지의 과정을 자동화하여, 연구와 발표 준비를 동시에 진행하는 연구자에게 매우 효율적입니다. Pandoc 명령을 통해 .md 파일이 reveal.js 기반의 HTML로 변환되고, 이 결과물이 GitHub Pages를 통해 즉시 공유됩니다.

요약 및 핵심 정리

  • 효율성과 일관성: 마크다운은 콘텐츠와 디자인을 분리하여 생산성을 높입니다.
  • Pandoc: 마크다운을 PPTX, Reveal.js 등 다양한 포맷으로 변환하는 핵심 엔진입니다.
  • Reveal.js: 프래그먼트, 코드 하이라이팅 등 기술 발표에 강력한 기능을 제공합니다.
  • 템플릿 활용: --reference-doc 옵션으로 PPTX 디자인 일관성을 유지할 수 있습니다.
  • 자동화: CI/CD와 연동하여 작성부터 배포까지의 과정을 자동화할 수 있습니다.
오늘 우리는 마크다운을 활용한 슬라이드 제작이 왜 효율적인지, 그리고 Pandoc, Reveal.js, Marp 같은 도구들을 어떻게 연동해야 하는지 살펴보았습니다. 핵심은 콘텐츠 작성과 디자인을 분리하고, 자동화된 변환 도구를 통해 반복 작업을 최소화하는 것입니다. 이제 학습한 내용을 바탕으로 실제 발표 자료 제작 워크플로우를 구축해 보시기 바랍니다.
